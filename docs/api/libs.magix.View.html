Class libs.magix.View

Magix View基类.继承自Backbone.View.用于管理View声明周期,事件代理,渲染数据,以及响应Hash变化.
MagixView子类位于app/views目录之下,通过重写init,render,queryModelChange,renderer,events来扩展出具体子类,如:
define(function(){
    require("libs/magix/view").extend({init:...,render:...});
}
通常不会直接通过new View(config)来创建类的实例,而是通过vcElement.mountView("viewName")的方式将view装载入一个容器(vcelement)时进行实例化.

Methods

init

void init ( )
实例化过程会调用init方法,在这个方法中可以完成一些初始化任务,比如载入将要使用的Model/Collection对象

queryModelChange

Boolean|Array queryModelChange ( )
query发生变化事件响应函数,基类中是个空方法,各子类在需要时实现这个方法
注意,并非query发生变化一定会触发这个方法,需要由父View决定是否将change事件下发给子View.
相应的return true将会下发change事件给子view,false反之,return vcid数组,将事件传递给指定的子View.

render

void render ( )
view渲染方法,各子类可以覆盖
默认会将query交给同名的模板文件,渲染至vcelement内.

setData

void setData ( )
设置数据到this.data,内部会自动把renderer附加到data中
mustahce.tohtml(template,this.setData({list:...}));
